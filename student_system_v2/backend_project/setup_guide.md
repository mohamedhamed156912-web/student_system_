# دليل تثبيت وتشغيل المشروع (Setup Guide)

لإعداد وتشغيل هذا المشروع على جهاز كمبيوتر (لابتوب) آخر، اتبع الخطوات التالية:

## 1. نقل ملفات المشروع (Copy Files)
قم بنسخ مجلد المشروع بالكامل إلى الجهاز الجديد. تأكد من الحفاظ على بنية المجلدات كما هي (ملف index.html ومجلد js وباقي الملفات).

## 2. الحاجة إلى خادم محلي (Local Server Required)
بما أن المشروع يستخدم تقنية **ES Modules** (وحدات JavaScript)، لا يمكن تشغيله بمجرد النقر المزدوج على ملف index.html في المتصفح، لأن المتصفحات تمنع تحميل هذه الملفات لأسباب أمنية (CORS Policy). يجب تشغيل المشروع عبر محاكي خادم محلي.

يمكنك القيام بذلك بإحدى الطرق التالية:

### الخيار أ: استخدام Node.js (الطريقة الأسرع)
إذا كان Node.js مثبتاً على الجهاز الجديد، افتح سطر الأوامر (Command Prompt أو PowerShell) داخل مجلد المشروع وقم بتشغيل الأمر التالي:
```bash
npx serve
```
بعد ذلك، افتح المتصفح وانتقل إلى الرابط: `http://localhost:3000`

### الخيار ب: استخدام إضافة "Live Server" في VS Code
١. افتح مجلد المشروع باستخدام برنامج **Visual Studio Code**.
٢. اذهب إلى قسم الإضافات (Extensions) وابحث عن **Live Server** وقم بتثبيتها.
٣. انقر بزر الفأرة الأيمن على ملف index.html واختر **Open with Live Server**.

### الخيار ج: استخدام Python
إذا كان Python مثبتاً، افتح سطر الأوامر في مجلد المشروع وقم بتشغيل:
```bash
python -m http.server
```
ثم انتقل إلى: `http://localhost:8000`

## 3. نقل البيانات والمستخدمين (Data Migration)
المشروع يخزن البيانات (الطلاب، الامتحانات، النتائج) محلياً داخل المتصفح (**IndexedDB**). لنقل البيانات التي قمت بإدخالها مسبقاً من جهاز لآخر:

1. **على الجهاز القديم:**
   - ادخل بصفة مسؤول (Admin) إلى لوحة التحكم.
   - اذهب إلى تبويب **إدارة النظام والبيانات** (System Management).
   - اضغط على زر **تصدير البيانات** (Export Data) وسيقوم بتنزيل ملف بصيغة `.json`.

2. **على الجهاز الجديد:**
   - بعد تشغيل المشروع بالطريقة المذكورة أعلاه.
   - ادخل إلى لوحة التحكم (Admin).
   - اذهب إلى تبويب **إدارة النظام والبيانات**.
   - اضغط على زر **استيراد البيانات** (Import Data) واختر الملف الذي قمت بتنزيله من الجهاز القديم.

> [!TIP]
> لا تحتاج لتثبيت أي قواعد بيانات خارجية (مثل SQL)؛ المتصفح يقوم بكل العمل تلقائياً بمجرد تشغيل المشروع كما هو موضح أعلاه.
