# 🎓 نظام طلاب الدلتا — Backend Setup Guide

---

## ⚡ متطلبات التشغيل
- **Node.js** v16 أو أحدث → https://nodejs.org
- لا يوجد متطلبات أخرى، قاعدة البيانات SQLite تتولد تلقائياً

---

## 🚀 خطوات التشغيل (3 خطوات فقط)

### الخطوة 1 — فتح Terminal في مجلد المشروع
```bash
cd student-system-backend
```

### الخطوة 2 — تثبيت المكتبات (مرة واحدة فقط)
```bash
npm install
```

### الخطوة 3 — تشغيل السيرفر
```bash
npm start
```

ثم افتح المتصفح على: **http://localhost:3000**

## 🔐 شروط كلمة المرور (جديد)
عند إنشاء حساب أو تغيير كلمة المرور، يجب أن تحتوي على:
- ✅ 8 أحرف على الأقل
- ✅ حرف كبير واحد على الأقل (A-Z)
- ✅ حرف صغير واحد على الأقل (a-z)
- ✅ رقم واحد على الأقل (0-9)

**مثال كلمة مرور صحيحة:** `Delta2024`

---

## 🗃️ قاعدة البيانات
- **النوع:** SQLite (ملف واحد: `database.sqlite`)
- يتم إنشاء الملف تلقائياً عند أول تشغيل
- **البيانات محفوظة على السيرفر** (مش في المتصفح زي قبل)

### جداول قاعدة البيانات:
| الجدول | المحتوى |
|--------|---------|
| `users` | المستخدمين (الطلاب والمدراء) |
| `annual_results` | النتائج السنوية لكل طالب |
| `exams` | الاختبارات |
| `questions` | أسئلة كل اختبار |
| `exam_results` | نتائج الطلاب في الاختبارات |

## 🌐 API Endpoints
| Method | URL | الوظيفة |
|--------|-----|---------|
| POST | /api/auth/login | تسجيل الدخول |
| POST | /api/auth/signup | إنشاء حساب جديد |
| POST | /api/auth/change-password | تغيير كلمة المرور |
| GET | /api/users/me | بيانات المستخدم الحالي |
| GET | /api/exams | الاختبارات المتاحة |
| POST | /api/exam-results | تسليم اختبار |
| GET | /api/admin/users | (مدير) كل الطلاب |
| GET | /api/admin/exams | (مدير) كل الاختبارات |

## ✅ ما الذي تغير؟
- ✅ **قاعدة بيانات SQLite حقيقية** على السيرفر (بدل IndexedDB في المتصفح)
- ✅ **كلمات المرور محمية** بـ bcrypt hashing
- ✅ **JWT Tokens** لأمان الجلسات
- ✅ **شروط كلمة المرور** مع مؤشر بصري مباشر
- ✅ **التصميم لم يتغير** إطلاقاً
- ✅ **طريقة فتح الصفحات لم تتغير** إطلاقاً
